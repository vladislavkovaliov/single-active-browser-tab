# 🧩 Single Active Browser Tab — Heartbeat Mechanism

## 1. Overview

Необходимо реализовать механизм, который гарантирует, что **только одна вкладка браузера может быть активной одновременно**.

Если пользователь открывает приложение во второй вкладке:

- Вторая вкладка должна определить, что уже существует активная вкладка
- И перейти в режим `blocked`
- Либо (опционально) иметь возможность выполнить takeover

Решение должно быть устойчивым к:

- Крашу вкладки
- Принудительному закрытию браузера
- Потере фокуса

---

## 2. Технические ограничения

### ❗ ВАЖНО

- Решение **не должно зависеть от React**
- Не должно зависеть от Vue, Angular или любого другого фреймворка
- Это должен быть **чистый JavaScript (ES6+)**
- Должно работать в браузере
- Не использовать сторонние библиотеки

Решение должно быть:

- Самодостаточным
- Легко подключаемым в любой SPA или MPA
- Возможным для использования как отдельный модуль

---

## 3. Основная логика

### 3.1 Идентификация вкладки

При загрузке страницы:

- Генерируется уникальный `tabId`
- Формат: `Date.now() + "-" + Math.random()`
- Пример: `"1700000000000-0.123456"`

---

### 3.2 Хранение состояния

Используется `localStorage`.

Ключ:

```
single-active-tab
```

Структура значения:

```json
{
  "id": "tab-id",
  "lastSeen": 1700000000000
}
```

Где:

- `id` — идентификатор вкладки
- `lastSeen` — timestamp последнего heartbeat

---

### 3.3 Проверка при открытии

При инициализации:

1. Считать данные из `localStorage`
2. Если записи нет → стать активной вкладкой
3. Если запись есть:
   - Проверить `Date.now() - lastSeen`
   - Если меньше `STALE_TIMEOUT` → вкладка считается живой → текущая вкладка блокируется
   - Если больше → вкладка считается мёртвой → текущая вкладка перехватывает управление

---

## 4. Heartbeat

### Интервал:

```
HEARTBEAT_INTERVAL = 2000ms
```

### Таймаут устаревания:

```
STALE_TIMEOUT = 5000ms
```

### Поведение:

- Активная вкладка обновляет `lastSeen` каждые `HEARTBEAT_INTERVAL`
- Если вкладка закрывается — она удаляет ключ из `localStorage`
- Если вкладка падает — через `STALE_TIMEOUT` её можно заменить

---

## 5. События и реакция

Механизм должен:

- Подписываться на `window.addEventListener("storage")`
- Реагировать на изменение ключа
- Если обнаружена другая активная вкладка — переходить в режим `blocked`

---

## 6. API модуля

Модуль должен предоставлять следующий интерфейс:

```js
const manager = createSingleTabManager(options)
```

### Опции:

```js
{
  key: string, // ключ localStorage
  heartbeatInterval: number,
  staleTimeout: number,
  onBlocked: function,
  onActive: function
}
```

---

### Методы:

```js
manager.start()
manager.stop()
manager.isActive()
manager.takeover() // опционально
```

---

## 7. Поведение состояний

### Active state

- Вкладка владеет lock
- Обновляет heartbeat
- Может выполнять бизнес-логику

### Blocked state

- Вкладка не владеет lock
- Не обновляет heartbeat
- Ожидает освобождения

---

## 8. Требования к устойчивости

Система должна корректно работать при:

- Быстром открытии 2 вкладок одновременно
- Одновременной записи в localStorage
- Краше активной вкладки
- Восстановлении вкладки через session restore

---

## 9. Дополнительные требования

- Код должен быть:
  - Модульным
  - Без глобальных переменных
  - Без утечек памяти
- Все таймеры должны очищаться
- Все слушатели событий должны удаляться при `stop()`
